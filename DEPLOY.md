# Deploying IPS Freight Platform (Vercel + Render)

This guide takes about **10 minutes** and results in a shareable URL anyone can open — no installation required on their end.

## Overview

| Service | What it hosts | Cost |
|---------|--------------|------|
| **Render** | Express API + PostgreSQL | Free tier |
| **Vercel** | React frontend | Free forever |

---

## Step 1 — Deploy the API on Render

1. Go to [render.com](https://render.com) and sign up / log in with your GitHub account.
2. From the dashboard click **New → Blueprint**.
3. Connect the **`Astec-Rate-Application`** GitHub repository.
4. Render reads `render.yaml` and auto-creates:
   - **ips-freight-api** (Express + Node 20)
   - **ips-freight-db** (PostgreSQL 16)
5. Click **Apply** — the first build takes ~3-5 minutes.
6. Once the deploy is green, open the **ips-freight-api** service and copy its URL.
   It looks like: `https://ips-freight-api.onrender.com`

> **Note:** The free Render API spins down after 15 minutes of inactivity. The first request after idle takes ~30 seconds to wake up — this is normal.

> **Note:** The free PostgreSQL database expires after **90 days**. Before expiry, upgrade to Render's $7/mo Starter database to keep your data, or export and re-import it.

---

## Step 2 — Deploy the Frontend on Vercel

1. Go to [vercel.com](https://vercel.com) and sign up / log in with your GitHub account.
2. Click **Add New → Project** and import the **`Astec-Rate-Application`** repository.
3. In the **Configure Project** screen:
   - Set **Root Directory** to `apps/web`
   - Under **Environment Variables**, add:
     | Name | Value |
     |------|-------|
     | `VITE_API_URL` | `https://ips-freight-api.onrender.com/api/v1` ← paste your Render URL |
4. Click **Deploy**. The build takes ~1-2 minutes.
5. Vercel gives you a URL like `https://ips-freight.vercel.app`. **This is the link you share.**

---

## Step 3 — Allow the Frontend to Call the API (CORS)

1. In Render, go to **ips-freight-api → Environment**.
2. Find **`CORS_ORIGINS`** and set it to your Vercel URL:
   ```
   https://ips-freight.vercel.app
   ```
3. Click **Save Changes** — Render redeploys automatically (~1 min).

---

## Step 4 — Verify

Open your Vercel URL in a browser. The dashboard should load. Open the browser's developer console (F12 → Network tab) and confirm API calls to `/api/v1/...` return 200 responses.

---

## Sharing the App

Send anyone this URL:
```
https://ips-freight.vercel.app
```
No installation, no Docker, no setup — just click and use.

---

## Updating the App

Every time you push to the `main` branch:
- Vercel automatically rebuilds and redeploys the frontend (~1 min).
- Render automatically rebuilds and redeploys the API (~3 min).

## Environment Variable Reference

### Render (API)

| Variable | Description | Required |
|----------|-------------|----------|
| `DATABASE_URL` | Set automatically by Render from the linked DB | ✅ Auto |
| `JWT_SECRET` | Random secret for signing JWTs (auto-generated by Render) | ✅ Auto |
| `CORS_ORIGINS` | Your Vercel frontend URL | ✅ Manual |
| `NODE_ENV` | `production` | ✅ Set |
| `API_PORT` | `10000` (Render's required port) | ✅ Set |
| `REDIS_URL` | Only needed if you add Redis later | Optional |

### Vercel (Frontend)

| Variable | Description | Required |
|----------|-------------|----------|
| `VITE_API_URL` | Full URL to your Render API e.g. `https://ips-freight-api.onrender.com/api/v1` | ✅ Manual |
